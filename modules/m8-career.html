CAREER

Você concluiu o py-cyber. E agora?

Você passou por 8 módulos cobrindo desde manipulação de filesystem até pipelines async com criptografia. O que diferencia analistas que crescem dos que estagnam não é o conteúdo aprendido — é a prática contínua e a curadoria de fontes confiáveis. Este módulo é seu mapa pra próxima fase.

Roadmap dos próximos 12 meses

01

Mês 1–2 · Consolidação

Refaça os 7 módulos em ambiente local (não browser). Adapte cada exemplo pro seu contexto real — logs do seu SOC, alertas da sua stack. Comite tudo num repositório privado seu.

02

Mês 3–4 · Projeto real

Escolha UMA dor do seu dia: triagem manual, relatório semanal, enrichment lento. Automatize com Python. Meça antes/depois. Apresente pra equipe.

03

Mês 5–6 · CTF + leitura técnica

1 CTF por mês (HackTheBox, TryHackMe, picoCTF). Leia 2 livros da lista abaixo. Comece a escrever writeups — mesmo que ninguém leia, cristaliza o aprendizado.

04

Mês 7–9 · Especialização

Escolha um vertical: AppSec, Threat Hunting, Incident Response, Cloud Security, Red Team. Vá fundo em UM. Generalista júnior é commodity; especialista é raro.

05

Mês 10–12 · Certificação + portfólio público

Tire UMA certificação relevante pra sua trilha. Publique 3 projetos no GitHub público. Escreva no LinkedIn ou blog. Sua próxima vaga vai encontrar você, não o contrário.

Livros que valem o tempo

  • PYTHONBlack Hat Python — Justin Seitz & Tim Arnold. O clássico de ofensiva em Python.
  • PYTHONViolent Python — TJ O'Connor. Mais antigo mas ainda formativo.
  • FORENSICSThe Art of Memory Forensics — Ligh, Case, Levy & Walters. Bíblia de Volatility.
  • IRIncident Response & Computer Forensics — Luttgens, Pepe, Mandia.
  • APPSECThe Web Application Hacker's Handbook — Stuttard & Pinto.
  • DETECTIONPractical Threat Intelligence — Valentina Costa-Gazcón.
  • DESIGNDesigning Secure Software — Loren Kohnfelder.
  • PYTHONFluent Python (2ª ed) — Luciano Ramalho. Pra dominar a linguagem de verdade.

Plataformas de prática hands-on

Hábitos que separam júnior de pleno

  • Leia código alheio — clone projetos open source de DFIR (Velociraptor, GRR, TheHive) e leia.
  • Documente o que você faz — runbooks pra você mesmo daqui a 6 meses.
  • Automatize o tédio — qualquer tarefa manual feita 3+ vezes vira script.
  • Saiba comunicar com não-técnicos — escrever bem é multiplicador de carreira.
  • Mantenha um lab pessoal — VM com Splunk free, ELK, Suricata. Quebre coisas em casa.
  • Saiba quando NÃO usar Python — bash às vezes é melhor, Go é melhor pra tooling distribuído, Rust pra perf.

Última recomendação

O melhor profissional de segurança que existe é o que continua curioso. Ferramentas mudam, vendor muda, certificação envelhece — mas a curiosidade técnica dobra de valor a cada ano. Continue quebrando coisas em laboratório, continue lendo writeups, continue escrevendo código.

Python é um meio. Segurança é um meio. O fim é proteger usuários, sistemas e dados de quem quer fazer mal a eles. Mantém isso no horizonte.
