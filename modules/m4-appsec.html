YAML — o formato preferido das misconfigs

Use sempre yaml.safe_load. O load sem safe permite executar código Python arbitrário (CVE-2017-18342, e muitas outras). 

PYTHONimport yaml

manifest = """
apiVersion: v1
kind: Pod
metadata:
  name: webapp
spec:
  containers:
    - name: api
      image: nginx:latest
      securityContext:
        privileged: true
        runAsUser: 0
"""

doc = yaml.safe_load(manifest)

# Auditoria: containers privilegiados são red flag
for c in doc["spec"]["containers"]:
    sc = c.get("securityContext", {})
    if sc.get("privileged"):
        print(f"⚠ {c['name']}: privileged=True")
    if sc.get("runAsUser") == 0:
        print(f"⚠ {c['name']}: rodando como root")
⚠ NUNCA USE yaml.load() Sem Loader=SafeLoader ou safe_load, um YAML malicioso pode executar código no momento do parsing. Mesmo que o YAML venha de fonte "confiável", trate como input não-confiável.

HTML — BeautifulSoup pra scraping defensivo

PYTHONfrom bs4 import BeautifulSoup

html = """
<html><body>
  <h1>Admin Panel</h1>
  <a href="/users">Users</a>
  <a href="/config">Config</a>
  <form action="/login" method="POST">
    <input name="user">
    <input name="pass" type="password">
  </form>
  <script>var token = "leaked_jwt_xyz"</script>
</body></html>
"""

soup = BeautifulSoup(html, "html.parser")

# Extrair endpoints de uma página
links = [a["href"] for a in soup.find_all("a", href=True)]

# Listar formulários e seus campos
for form in soup.find_all("form"):
    inputs = [i.get("name") for i in form.find_all("input")]
    print(form["action"], form["method"], inputs)

# Achar segredos vazados em script tags
import re
for s in soup.find_all("script"):
    if s.string and re.search(r"token|api[_-]?key|secret", s.string, re.I):
        print("⚠ possível segredo:", s.string)

XML — relatórios de scanner

OWASP ZAP, Nuclei, Nessus, Nmap — todos exportam XML. Use xml.etree da stdlib (sem dependências) ou lxml (mais rápido, suporta XPath). 

PYTHONimport xml.etree.ElementTree as ET

zap_xml = """<?xml version="1.0"?>
<OWASPZAPReport>
  <site name="https://app.local">
    <alerts>
      <alertitem>
        <name>SQL Injection</name>
        <riskcode>3</riskcode>
        <url>https://app.local/search?q=1</url>
      </alertitem>
      <alertitem>
        <name>Missing X-Frame-Options</name>
        <riskcode>1</riskcode>
        <url>https://app.local/</url>
      </alertitem>
    </alerts>
  </site>
</OWASPZAPReport>"""

root = ET.fromstring(zap_xml)

# Filtrar só achados de alto risco (riskcode >= 3)
for alert in root.iter("alertitem"):
    risk = int(alert.findtext("riskcode", "0"))
    if risk >= 3:
        print(alert.findtext("name"), "→", alert.findtext("url"))
⚠ XXE — XML EXTERNAL ENTITY xml.etree da stdlib é seguro por padrão (não resolve entidades externas). Mas se for usar lxml ou xml.dom.minidom, configure resolve_entities=False e no_network=True. Veja a lib defusedxml pra parsing 100% seguro.

Hashing — integridade de artefato

PYTHONimport hashlib

def file_hash(content: bytes, algo="sha256") -> str:
    h = hashlib.new(algo)
    h.update(content)
    return h.hexdigest()

amostra = b"binary_payload_evidencia"
file_hash(amostra, "sha256")
file_hash(amostra, "md5")

# Comparar com IOC conhecido (chain of custody)
ioc_hash = "5d41402abc4b2a76b9719d911017c592"
if file_hash(amostra, "md5") == ioc_hash:
    print("MATCH com IOC conhecido")

Bandit — SAST embutido no fluxo

Bandit é o linter de segurança pra Python. Encontra eval(), pickle.load, hardcoded passwords, subprocess com shell=True, uso de MD5 fraco, etc. CLI normal: 

BASH# Local — não roda no Pyodide, mas é o fluxo padrão
pip install bandit
bandit -r ./meu_projeto/
bandit -f json -o report.json ./meu_projeto/

Como Pyodide não tem bandit pré-empacotado, mostramos o que ele detectaria de forma manual — útil pra entender as regras: 

PYTHONimport re

codigo_suspeito = '''
import pickle, hashlib, subprocess
password = "admin123"
data = pickle.loads(user_input)
h = hashlib.md5(b"x")
subprocess.call(cmd, shell=True)
eval(expr)
'''

regras = {
    "B105_hardcoded_password": r"password\s*=\s*[\"'][^\"']+[\"']",
    "B301_pickle": r"pickle\.loads?\(",
    "B303_md5": r"hashlib\.md5\(",
    "B602_shell_true": r"shell\s*=\s*True",
    "B307_eval": r"\beval\s*\(",
}

for rule, pat in regras.items():
    if re.search(pat, codigo_suspeito):
        print(f"⚠ {rule}")
DICA DE CAMPO Em pipeline CI/CD: rode bandit -ll (só medium+ high) pra não estourar o pipeline com noise. --exit-zero no início pra reportar sem quebrar build, depois aperta.
