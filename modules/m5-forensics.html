SOC & CSIRT Forensics

Timeline forense, regex de IOC, hashing de evidência, criptografia e resposta a incidente.

RESPONSE

O dia a dia do CSIRT

Resposta a incidente é triagem em ritmo de relógio. Você recebe um alerta, precisa: extrair IOCs do log, hashear evidência pra cadeia de custódia, construir timeline cruzando fontes, decidir se isola o host. Python é a cola entre todas essas tarefas.

CASOS REAIS
  • Extrair IPs/hashes/URLs de um log
  • Construir timeline de múltiplas fontes
  • Hashear bucket de evidência
  • Decifrar payload C2 simples
  • Normalizar timestamps em UTC
STACK DESTE MÓDULO
  • re — extração de IOCs
  • datetime — timeline
  • dateutil — parser flexível
  • hashlib — chain of custody
  • cryptography — Fernet/AES
⚠ FERRAMENTAS QUE NÃO RODAM NO BROWSER scapy (pcap), paramiko (SSH), volatility3 (memory), pefile (binários) — todos exigem Python local. Aqui foco no que dá pra exercitar no browser: regex, datetime, hashing, criptografia simétrica.

IOC extraction com regex

PYTHONimport re

evento = """
2024-10-12T09:14:01Z host=srv01 user=admin
src_ip=45.33.32.156 dst_ip=10.0.0.42
url=http://malicious.example.com/payload.exe
hash=5d41402abc4b2a76b9719d911017c592
sha256=e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855
domain=evil.example.org
"""

PATTERNS = {
    "ipv4": r"\b(?:\d{1,3}\.){3}\d{1,3}\b",
    "md5": r"\b[a-fA-F0-9]{32}\b",
    "sha256": r"\b[a-fA-F0-9]{64}\b",
    "url": r"https?://[^\s]+",
    "domain": r"\b[a-z0-9-]+\.[a-z]{2,}(?:\.[a-z]{2,})?\b",
}

def extract_iocs(text: str) -> dict:
    return {k: list(set(re.findall(p, text))) for k, p in PATTERNS.items()}

extract_iocs(evento)

Timestamps — sempre UTC, sempre ISO 8601

Cada fonte usa um formato diferente. A regra de ouro: parse na entrada, converte pra UTC, armazena como ISO 8601. Comparações ficam triviais. 

PYTHONfrom datetime import datetime, timezone
from dateutil import parser as dtparser

# Diferentes fontes, formatos misturados
fontes = [
    "2024-10-12T09:14:01Z",            # SIEM (ISO)
    "Oct 12 09:14:01",                # syslog
    "12/Oct/2024:09:14:01 -0300",     # Apache
    "1728724441",                     # epoch
]

def to_utc(s):
    if s.isdigit():
        return datetime.fromtimestamp(int(s), tz=timezone.utc)
    dt = dtparser.parse(s)
    if dt.tzinfo is None:
        dt = dt.replace(tzinfo=timezone.utc)
    return dt.astimezone(timezone.utc)

for f in fontes:
    print(f, "→", to_utc(f).isoformat())

Timeline — ordenando eventos de fontes mistas

PYTHONfrom dateutil import parser as dtparser

eventos = [
    {"src": "firewall", "ts": "2024-10-12T09:14:01Z", "msg": "DENY 45.33.32.156"},
    {"src": "sshd",     "ts": "Oct 12 09:14:03",      "msg": "Failed root login"},
    {"src": "edr",      "ts": "2024-10-12T09:13:55Z", "msg": "powershell.exe spawned"},
    {"src": "proxy",    "ts": "2024-10-12T09:14:08Z", "msg": "GET evil.example.org"},
]

for e in eventos:
    e["_dt"] = dtparser.parse(e["ts"])

timeline = sorted(eventos, key=lambda e: e["_dt"])
for e in timeline:
    print(f"{e['_dt'].isoformat()} [{e['src']:8}] {e['msg']}")

Hashing — chain of custody

PYTHONimport hashlib, json
from datetime import datetime, timezone

def custody_record(evidence: bytes, label: str) -> dict:
    return {
        "label": label,
        "size": len(evidence),
        "md5": hashlib.md5(evidence).hexdigest(),
        "sha1": hashlib.sha1(evidence).hexdigest(),
        "sha256": hashlib.sha256(evidence).hexdigest(),
        "collected_at": datetime.now(timezone.utc).isoformat(),
    }

evidencia = b"dump_de_memoria_ou_arquivo_suspeito"
record = custody_record(evidencia, "hd_imagem_01")
print(json.dumps(record, indent=2))
DICA DE CAMPO Toda evidência ganha 3 hashes (MD5, SHA1, SHA256) e timestamp UTC no momento da coleta. MD5 segue presente porque ferramentas legadas/AVs usam — não pra segurança, pra compatibilidade de IOC.

Criptografia simétrica — Fernet

Fernet (do cryptography) é AES-128-CBC + HMAC-SHA256 empacotado de forma segura por padrão. Use pra cifrar artefato sensível antes de enviar pra storage compartilhado. 

PYTHONfrom cryptography.fernet import Fernet

# Gera chave (guarde em vault, nunca em código!)
key = Fernet.generate_key()
cipher = Fernet(key)

# Cifrar relatório de incidente
relatorio = b"IR-2024-042: lateral movement detectado em srv01"
token = cipher.encrypt(relatorio)
print("cifrado:", token[:40], "...")

# Decifrar
plaintext = cipher.decrypt(token)
print("plaintext:", plaintext)

Hash de senhas — nunca cifre, hash com salt

PYTHONimport hmac, hashlib

# HMAC — assinar payload pra detectar tampering
# (em produção, use bcrypt/argon2 pra senhas — não rodam no Pyodide)
secret = b"chave_compartilhada_do_vault"
payload = b"IR-2024-042:status=contained"

assinatura = hmac.new(secret, payload, hashlib.sha256).hexdigest()
print(assinatura)

# Verificação resistente a timing attack
recebido = assinatura
hmac.compare_digest(assinatura, recebido)
⚠ NUNCA MD5/SHA1 puro pra senha. Sempre pbkdf2_hmac, bcrypt, scrypt ou argon2 com salt aleatório por usuário e centenas de milhares de iterações.

Desafio do módulo

Construa triage_incident(log_text) que: (1) extrai todos os IOCs (IPs, hashes, URLs, domínios) com regex, (2) parseia todos os timestamps que encontrar e ordena cronologicamente, (3) gera registro de cadeia de custódia com hash do log, (4) cifra o resultado final com Fernet e devolve (token, key) pronto pra envio seguro.
