File System & OS Manipulation

Manipule arquivos, processos e o filesystem programaticamente — base de toda automação forense e de cleanup.
CORE

Por que isso importa em segurança

Resposta a incidente é, em essência, três coisas: encontrar arquivos, ler arquivos e copiar arquivos com integridade. Coletar artefatos de /var/log, varrer um diretório atrás de IOCs, listar processos suspeitos, mover evidência para um bucket forense — tudo passa pelo módulo os, pathlib e shutil.

CASOS REAIS
  • Coletar todos os .log dos últimos 7 dias
  • Hashear todos arquivos de um diretório suspeito
  • Listar processos com nome estranho
  • Copiar evidência preservando metadata
  • Criar diretórios de quarentena
STDLIB QUE VOCÊ USA
  • pathlib — caminhos modernos, OO
  • os — variáveis de ambiente, PIDs
  • shutil — copiar, mover, comprimir
  • glob — busca por padrão
  • sys — argv, exit codes

pathlib — esqueça os.path.join()

pathlib.Path é a forma moderna e cross-platform. Ele opera com operador / em vez de concatenação manual.

PYTHONfrom pathlib import Path

# Diretório atual
cwd = Path.cwd()

# Construir caminhos com /
log_dir = Path("/var/log") / "app" / "errors.log"

# Propriedades úteis
log_dir.name        # "errors.log"
log_dir.suffix      # ".log"
log_dir.parent      # Path("/var/log/app")
log_dir.exists()    # bool
log_dir.is_file()   # bool

# Listar arquivos de um diretório
[p for p in Path(".").iterdir() if p.is_file()]

Buscar arquivos — glob recursivo

Para forense, rglob (recursivo) é o pão-com-manteiga. Encontra todos os arquivos que batem um padrão a partir de um diretório raiz.

PYTHONfrom pathlib import Path

# Todos os .log abaixo de /var/log
logs = list(Path("/var/log").rglob("*.log"))

# Múltiplos padrões — caso forense
suspicious = []
for ext in ["*.exe", "*.dll", "*.ps1"]:
    suspicious += list(Path("/tmp").rglob(ext))

⚠ NUNCA Faça os.environ["KEY"] direto sem default — levanta KeyError se a variável não existir e quebra o script. Use sempre os.getenv("KEY", default).

shutil — copiar com integridade

Em forense, copiar mantendo metadata (mtime, permissões) é obrigatório. Use shutil.copy2, nunca copy.

PYTHONimport shutil
from pathlib import Path

# Copiar preservando metadata
shutil.copy2("/var/log/auth.log", "/forensics/case-042/")

# Comprimir um diretório inteiro (evidência)
shutil.make_archive("case-042-evidence", "zip", "/forensics/case-042")

# Espaço em disco (antes de coletar)
total, used, free = shutil.disk_usage("/")

Lendo arquivos do jeito certo

Sempre with open(...) — context manager fecha o handle mesmo se der exceção. Para logs grandes, nunca use .read() — itere linha a linha.

PYTHON# RUIM — carrega 2GB de log na RAM
content = open("/var/log/huge.log").read()

# BOM — streaming, memória constante
with open("/var/log/huge.log") as f:
    for line in f:
        if "FAILED" in line:
            print(line.strip())

# Pathlib tem atalhos:
from pathlib import Path
texto = Path("config.yaml").read_text()
linhas = Path("ips.txt").read_text().splitlines()
💡 PADRÃO SOC Para parsing de logs em produção use pathlib + iteração linha a linha + regex compilada. Isso processa GBs sem estourar RAM.

Sandbox WASM — o que esperar

O Pyodide tem um filesystem virtual (MEMFS) — você pode criar arquivos, escrever, ler, mas nada persiste entre sessões. Use isso a seu favor para praticar:

PYTHONfrom pathlib import Path

# Criar diretório no sandbox
Path("/tmp/lab").mkdir(exist_ok=True)

# Escrever um log fake para praticar parsing
Path("/tmp/lab/auth.log").write_text(
    "Failed password for root from 10.0.0.5\n"
    "Accepted password for ana from 10.0.0.10\n"
)

# Ler de volta
Path("/tmp/lab/auth.log").read_text()
