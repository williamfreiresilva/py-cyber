Data & Log Analysis

Pandas como planilha programável do analista de SOC — ingestão, filtros, agregações e relatórios em XLSX.

DATA

Por que pandas no SOC?

Logs viraram tabelas. Export do SIEM, alerta do EDR, feed de IOCs do MISP, CSV do Splunk — tudo cai como linha × coluna. Pandas transforma essa massa em consultas tipo SQL, com a flexibilidade do Python por cima.

CASOS REAIS
  • Contar tentativas de login por IP
  • Juntar duas planilhas de IOCs
  • Detectar usuário em múltiplos hosts
  • Gerar XLSX para a gerência
  • Triagem de alertas por severidade
STACK DESTE MÓDULO
  • pandas — DataFrames
  • numpy — base numérica
  • openpyxl — XLSX
  • re — parsing de log

DataFrame: a estrutura central

Um DataFrame é uma tabela: colunas tipadas, índice de linhas, métodos para inspeção rápida.

iocs = {

Filtros e seleção

Boolean masks são o pão com manteiga. Você passa uma série de True/False entre colchetes e o DataFrame retorna só as linhas que casam. 

PYTHON# IOCs com score acima de 5
df[df["score"] > 5]

# Combinando condições — use & e | com parênteses
df[(df["score"] > 0) & (df["tag"] != "clean")]

# Sintaxe alternativa com query()
df.query("score > 5")

Group by — o coração da detecção

Agregar por chave é o que transforma log bruto em sinal. "Quantas vezes cada IP tentou logar?" é um groupby. 

PYTHONtentativas = pd.DataFrame({
    "ip": ["10.0.0.5", "10.0.0.5", "10.0.0.5",
           "192.168.1.7", "45.33.32.156", "45.33.32.156"],
    "status": ["fail", "fail", "ok", "ok", "fail", "fail"],
})

# Total de eventos por IP
tentativas.groupby("ip").size()

# Falhas por IP — ordenadas
falhas = tentativas[tentativas["status"] == "fail"]
falhas.groupby("ip").size().sort_values(ascending=False)

Parsing de auth.log com regex + pandas

Caso real: você recebeu um auth.log e precisa identificar brute-force SSH. Estratégia: regex extrai os campos, pandas consolida. 

PYTHONimport re
import pandas as pd

log = """Oct 12 09:14:01 srv sshd[1201]: Failed password for root from 45.33.32.156 port 51022 ssh2
Oct 12 09:14:03 srv sshd[1202]: Failed password for root from 45.33.32.156 port 51024 ssh2
Oct 12 09:14:05 srv sshd[1203]: Failed password for admin from 45.33.32.156 port 51026 ssh2
Oct 12 09:15:10 srv sshd[1210]: Accepted password for william from 10.0.0.42 port 60112 ssh2
Oct 12 09:16:22 srv sshd[1220]: Failed password for invalid user test from 185.220.101.5 port 44012 ssh2
Oct 12 09:16:24 srv sshd[1221]: Failed password for invalid user oracle from 185.220.101.5 port 44014 ssh2
Oct 12 09:16:26 srv sshd[1222]: Failed password for invalid user pi from 185.220.101.5 port 44016 ssh2
Oct 12 09:18:44 srv sshd[1230]: Accepted password for william from 10.0.0.42 port 60120 ssh2"""

padrao = re.compile(
    r"(?P<status>Failed|Accepted) password for (?:invalid user )?(?P<user>\S+) from (?P<ip>\S+)"
)

linhas = []
for linha in log.splitlines():
    m = padrao.search(linha)
    if m:
        linhas.append(m.groupdict())

eventos = pd.DataFrame(linhas)
eventos.head()

# Brute-force: IPs com 3+ falhas
falhas = eventos[eventos["status"] == "Failed"]
suspeitos = falhas.groupby("ip").size().reset_index(name="falhas")
suspeitos = suspeitos[suspeitos["falhas"] >= 3]
suspeitos
DICA DE CAMPO Em produção, esse padrão escala: troque a string log por Path("/var/log/auth.log").read_text() — o resto do código não muda.

CSV e JSON nativos

No Pyodide, o filesystem é virtual (MEMFS) — tudo vive em RAM. Você escreve e lê arquivos normalmente, mas eles somem ao recarregar a página. 

PYTHON# Escrever CSV
suspeitos.to_csv("/tmp/suspeitos.csv", index=False)

# Ler CSV de volta
de_volta = pd.read_csv("/tmp/suspeitos.csv")
de_volta

# JSON — útil para APIs (próximo módulo!)
eventos.to_json("/tmp/eventos.json", orient="records", indent=2)
pd.read_json("/tmp/eventos.json")

Relatório executivo com openpyxl

Gestor não abre CSV — gestor abre XLSX. Pandas usa openpyxl como engine por baixo dos panos. 

PYTHON# Salvar XLSX
suspeitos.to_excel("/tmp/relatorio.xlsx", index=False, sheet_name="BruteForce")

# Ler de volta para validar
pd.read_excel("/tmp/relatorio.xlsx")

# Múltiplas abas — relatório completo
with pd.ExcelWriter("/tmp/soc_report.xlsx") as w:
    eventos.to_excel(w, sheet_name="eventos", index=False)
    suspeitos.to_excel(w, sheet_name="suspeitos", index=False)
⚠ MEMFS É VOLÁTIL Arquivos em /tmp/ ou qualquer caminho do Pyodide vivem só na sessão do browser. Para baixar de verdade, é preciso boilerplate JS (js.Blob + URL.createObjectURL). Em ambiente real (Python local), basta salvar no disco.

Desafio do módulo

A partir do auth.log embutido acima, gere um XLSX com duas abas: eventos (parse completo) e brute_force (apenas IPs com ≥3 falhas, ordenado decrescente). Esse é o tipo de entregável que vai parar no e-mail do CISO.
