APIs & EDR/SIEM

Consumir REST, validar payloads e integrar com VirusTotal, CrowdStrike, QRadar.

INTEGRATION

Por que toda ferramenta de segurança tem API

VirusTotal, AbuseIPDB, MISP, CrowdStrike Falcon, QRadar, Splunk, Elastic — todos expõem REST. O analista que sabe consumir API automatiza enrichment, triagem e contenção sem depender de console. É o pulo do gato do SOC moderno.

CASOS REAIS
  • Enrichment de IOC no VirusTotal
  • Reputação de IP no AbuseIPDB
  • Query EDR para hosts comprometidos
  • Push de IOC pro firewall via API
  • Pull de alertas do SIEM em batch
STACK DESTE MÓDULO
  • requests — HTTP síncrono
  • pyodide.http — fetch no browser
  • pydantic — validação de payload
  • json — serialização
  • base64 — auth headers
⚠ AMBIENTE WASM No browser, requests bloqueante não funciona — usamos pyodide.http.pyfetch (async) ou urllib com patch. Em Python local, requests.get() funciona normalmente. Os exemplos aqui usam a API pública httpbin.org e api.github.com, que aceitam CORS.

HTTP básico com pyfetch

No Pyodide, requisições HTTP são assíncronas. pyfetch é o wrapper Python sobre o fetch do browser. 

PYTHONfrom pyodide.http import pyfetch

resp = await pyfetch("https://httpbin.org/get")
data = await resp.json()
data["url"]

GET com query params e headers

PYTHONfrom pyodide.http import pyfetch

url = "https://api.github.com/repos/python/cpython"
resp = await pyfetch(
    url,
    headers={"Accept": "application/vnd.github+json"}
)
repo = await resp.json()
print(repo["full_name"], repo["stargazers_count"])

POST com JSON body

PYTHONimport json
from pyodide.http import pyfetch

payload = {"ioc": "45.33.32.156", "type": "ip"}
resp = await pyfetch(
    "https://httpbin.org/post",
    method="POST",
    headers={"Content-Type": "application/json"},
    body=json.dumps(payload)
)
echo = await resp.json()
echo["json"]

Auth — Bearer e Basic

PYTHONimport base64

# Bearer (VirusTotal, CrowdStrike, Splunk)
headers = {"Authorization": "Bearer " + "SEU_TOKEN_AQUI"}

# Basic (QRadar legado, alguns CMDBs)
creds = base64.b64encode(b"user:pass").decode()
headers = {"Authorization": "Basic " + creds}

# API Key custom (VirusTotal v3)
headers = {"x-apikey": "SEU_VT_KEY"}

Pydantic — não confie em payload externo

API responde JSON. Você precisa garantir tipos antes de processar — IP precisa ser IP, score precisa ser inteiro. Pydantic faz isso declarativamente. 

PYTHONfrom pydantic import BaseModel, Field
from typing import Optional

class IOCReport(BaseModel):
    ip: str
    score: int = Field(ge=0, le=100)
    country: Optional[str] = None
    malicious: bool = False

# Valida e normaliza
raw = {"ip": "45.33.32.156", "score": "87", "malicious": 1}
report = IOCReport(**raw)
report.model_dump()

Caso real: enriquecer IOC com 2 fontes

PYTHONfrom pyodide.http import pyfetch
from pydantic import BaseModel

class Enrichment(BaseModel):
    ip: str
    geo_country: str = ""
    geo_org: str = ""

async def enrich(ip: str):
    r = await pyfetch(f"https://ipapi.co/{ip}/json/")
    j = await r.json()
    return Enrichment(
        ip=ip,
        geo_country=j.get("country_name", ""),
        geo_org=j.get("org", "")
    )

result = await enrich("8.8.8.8")
result.model_dump()
DICA DE CAMPO Em Python local, troque pyfetch por requests.get() e tire os await. O resto do código (pydantic, parsing, lógica) fica idêntico — é por isso que separar I/O de lógica importa.

Rate limit e retry

PYTHONimport asyncio
from pyodide.http import pyfetch

async def get_with_retry(url, max_retry=3):
    for i in range(max_retry):
        r = await pyfetch(url)
        if r.status == 200:
            return await r.json()
        if r.status == 429:
            await asyncio.sleep(2 ** i)
    raise RuntimeError(f"falhou após {max_retry} tentativas")

await get_with_retry("https://httpbin.org/status/200")

Desafio do módulo

Construa uma função triage(ip) que: (1) faz GET em ipapi.co/<ip>/json/, (2) valida o retorno com pydantic, (3) retorna um dict com ip, country, org e verdict (string "suspicious" se org contém "Tor" ou "VPN", senão "clean"). Teste com 3 IPs diferentes.
